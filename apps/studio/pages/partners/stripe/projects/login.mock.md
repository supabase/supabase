# Stripe Projects login mocks

These review-only URLs bypass the real Stripe Projects `ar_id` lookup so reviewers can see the
interstitial states without setting up the Stripe CLI flow.

## Local URLs

| State                       | URL                                                                       |
| --------------------------- | ------------------------------------------------------------------------- |
| New organization request    | `http://localhost:8082/partners/stripe/projects/login?mock=pending`       |
| Already-linked organization | `http://localhost:8082/partners/stripe/projects/login?mock=linked`        |
| Wrong account               | `http://localhost:8082/partners/stripe/projects/login?mock=wrong-account` |
| Success                     | `http://localhost:8082/partners/stripe/projects/login?mock=success`       |

The mocks are enabled outside production and on Vercel preview URLs. Production `supabase.com`
continues to require a real `ar_id`.

## Real flow

The production route still uses:

- `GET /platform/stripe/projects/provisioning/account_requests/{id}`
- `POST /platform/stripe/projects/provisioning/account_requests/{id}/confirm`

The `ar_id` is generated by Stripe Projects and passed to Studio in the query string.
