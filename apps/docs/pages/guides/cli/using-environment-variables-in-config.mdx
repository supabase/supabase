The Supabase CLI is capable of utilizing environment variables from `.env` file in our project's root directory.

```bash .env
GITHUB_CLIENT_ID=""
GITHUB_SECRET=""
```

We can reference the environment variable by using the `env()` function within the `supabase/config.toml` file.

```toml supabase/config.toml
[auth.external.github]
enabled = true
client_id = "env(GITHUB_CLIENT_ID)"
secret = "env(GITHUB_SECRET)"
# Overrides the default auth redirectUrl.
redirect_uri = ""
```

These values will be substituted at runtime by the CLI when running the `supabase start` command.
