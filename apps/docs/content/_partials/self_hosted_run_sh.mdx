<Admonition type="note">

These steps use `sh run.sh`, a convenience wrapper around `docker compose` included in the `docker/` directory. It reads any active override files from the `COMPOSE_FILE` variable in your `.env`, so you don't need to repeat `-f` flags. See [Managing the stack](/docs/guides/self-hosting/docker#managing-the-stack) for the full command reference and the underlying `docker compose` equivalents.

</Admonition>
