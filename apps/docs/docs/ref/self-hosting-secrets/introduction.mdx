    This page covers the required **secrets** and environment variables when self-hosting Supabase.
    These values are critical for security and proper functioning of your deployment.

    ---

    ## `SECRET_KEY_BASE`

    `SECRET_KEY_BASE` is used to sign and encrypt sensitive data (such as session cookies).
    It must be **long, random, and kept secret**.

    ### Generate a key

    You can generate a secure key using:

    ```bash
    openssl rand -hex 64
    ```

    **Example output:**
    ```
    3bKhs29sdfjskljfslkfj... (64 chars)
    ```

    ### Where to put it

    Update your `.env` (or `.env.example` if youâ€™re making a template) with:

    ```
    SECRET_KEY_BASE=your-generated-secret
    ```

    ### Can it be changed later?

    You can rotate `SECRET_KEY_BASE` by updating the value in your `.env` file and restarting the containers.
    However, doing so invalidates existing sessions, so all users will need to log in again.
