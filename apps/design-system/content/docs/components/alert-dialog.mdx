Alert Dialog interrupts the user’s workflow to communicate critical information or require an explicit decision (for example, “Are you sure you want to delete this?”).

<ComponentPreview name="alert-dialog-demo" peekCode wide />

Unlike a generic [Dialog](../components/dialog), it **cannot be dismissed by clicking outside** — the user must take an action (cancel, confirm, or press Escape).

**When to use**:

- Critical confirmations (delete, destructive actions)
- Important warnings that require acknowledgment
- Actions that cannot be undone
- Situations where you want a clear, explicit confirm/cancel choice

**Components**: `AlertDialog`, `AlertDialogContent`, `AlertDialogHeader`, `AlertDialogTitle`, `AlertDialogDescription`, `AlertDialogFooter`, `AlertDialogCancel`, `AlertDialogAction`, `AlertDialogTrigger` (from `ui`)

## Notes

- **Keep the description simple**: `AlertDialogDescription` renders as a `<p>` and cannot contain block-level children (like `<ul>`, `<div>`, multiple paragraphs, etc.). If you need richer content, use [Dialog](../components/dialog) or [Confirmation Modal](../fragments/confirmation-modal).
- **Always provide a cancel action**: include `AlertDialogCancel` in the footer so users can back out with a clear button (in addition to Escape).

See [Modality](../ui-patterns/modality) for guidance on choosing between dialog patterns.

## Usage

```tsx
import {
  AlertDialog,
  AlertDialogAction,
  AlertDialogCancel,
  AlertDialogContent,
  AlertDialogDescription,
  AlertDialogFooter,
  AlertDialogHeader,
  AlertDialogTitle,
  AlertDialogTrigger,
} from '@/components/ui/alert-dialog'
```

```tsx
<AlertDialog>
  <AlertDialogTrigger>Open</AlertDialogTrigger>
  <AlertDialogContent>
    <AlertDialogHeader>
      <AlertDialogTitle>Are you absolutely sure?</AlertDialogTitle>
      <AlertDialogDescription>
        This action cannot be undone. This will permanently delete your account and remove your data
        from our servers.
      </AlertDialogDescription>
    </AlertDialogHeader>
    <AlertDialogFooter>
      <AlertDialogCancel>Cancel</AlertDialogCancel>
      <AlertDialogAction>Continue</AlertDialogAction>
    </AlertDialogFooter>
  </AlertDialogContent>
</AlertDialog>
```
