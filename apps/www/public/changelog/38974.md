## Configure to save Auth Audit Logs in project's database

<img width="1182" height="369" alt="image" src="https://github.com/user-attachments/assets/0b0c434a-a37e-4d87-bbdf-2ff4caaa74f7" />

We've added a new page under the Auth section "Audit Logs" which will allow you to toggle disabling writing Auth audit logs to your project's database. The Auth audit logs will still be available in the [auth logs](https://supabase.com/dashboard/project/_/logs/auth-logs) but doing so will then prevent the audit logs from taking up your project's database space, which subsequently reduces your database storage costs. Read more about this in our docs [here](https://supabase.com/docs/guides/auth/audit-logs)!

PR: [https://github.com/supabase/supabase/pull/37409](https://github.com/supabase/supabase/pull/37409)
