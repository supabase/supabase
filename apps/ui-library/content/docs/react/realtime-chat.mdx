## Introduction

The Realtime Chat component provides a fully functional chat interface that syncs messages across all users in real-time using Supabase's real-time features. It includes:

- Real-time message syncing
- User identification
- Message history
- Input field for sending messages
- Visual distinction between own and other users' messages

## Usage

```tsx
import { RealtimeChat } from '@/registry/default/blocks/realtime-chat/components/realtime-chat'

export default function ChatRoom() {
  return (
    <RealtimeChat
      roomName="general"
      username="john_doe"
      avatar="https://example.com/avatar.jpg" // optional
    />
  )
}
```

## Props

| Prop     | Type              | Description                                                                       |
| -------- | ----------------- | --------------------------------------------------------------------------------- |
| roomName | string            | The name of the chat room. Users in the same room will see each other's messages. |
| username | string            | The display name for the current user.                                            |
| avatar   | string (optional) | URL for the user's avatar image.                                                  |

## Database Setup

This component requires a Supabase table called `messages` with the following structure:

```sql
create table messages (
  id uuid default uuid_generate_v4() primary key,
  room text not null,
  content text not null,
  user jsonb not null,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- Enable realtime for the messages table
alter publication supabase_realtime add table messages;
```

## Styling

The component uses Tailwind CSS for styling and can be customized using the following classes:

- Messages container: `flex-1 overflow-y-auto p-4 space-y-4`
- Input container: `border-t p-4`
- Message input: `w-full px-4 py-2 border rounded-lg`

You can override these styles by passing custom className props to the component.
